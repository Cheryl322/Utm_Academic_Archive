# 🗄️ Database Systems

This repository documents my structured query languages (SQL) scripts, entity normalization workflows, and relational data architecture blueprints.

---

## 📝 1. Course Summary & Gained Knowledge
This course established my capabilities in relational database architecture design, database normalization theory, and query execution management under enterprise-scale transactional constraints.

Core relational database structures mastered include:
*   **Relational Schema Normalization:** Translating raw information inputs and anomalies into normalized relational tables up to strict Third Normal Form (3NF) boundaries.
*   **Entity-Relationship Mapping:** Transforming complex corporate requirements data into robust Entity-Relationship Diagrams (ERD) while managing strict integrity rules.
*   **Structured Query Compiling:** Writing relational SQL scripts using advanced joins, aggregated functions layers, subqueries, and table manipulation queries.

## 🛠️ 2. Actionable Suggestions for Improvement
To bring the database management workflows up to modern backend infrastructure guidelines:
*   **Document Model Exploration:** Introducing a brief, comparative lab segment using an unstructured document store model (such as NoSQL MongoDB) would broaden our perspectives on horizontal cluster scaling.

## 🧠 3. Advanced Engineering Reflection & Creativity (Creative Insights)
A permanent engineering philosophy I adopted this term is that system resilience depends entirely on database integrity. Normalizing data layers proved that structural precision is the ultimate firewall against system synchronization failures. My creative system design workflow now locks relational integrity rules directly at the schema constraints level to guarantee absolute security before writing a single line of application logic.
