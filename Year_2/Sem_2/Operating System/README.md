# ⚙️ Operating System

This directory maps my processor scheduling simulations, thread synchronization rules evaluations, and low-level kernel resource tracking scripts.

## 📝 1. Course Summary & Gained Knowledge
This module rounded out my computer science baseline by uncovering the foundational management layers that monitor execution schedules, memory allocation bounds, and thread concurrency inside modern hardware architectures.

Key hardware-software synchronization mechanisms analyzed include:
* **Processor Scheduling Simulations:** Modeling mathematical runtime performance metrics (like Turnaround and Waiting times) across CPU task allocators (FCFS, Shortest Job First, Round Robin).
* **Concurrency Race Concurrency Checks:** Evaluating thread synchronization locks, processing state semaphores, and debugging deadlocks inside shared memory scopes.
* **Virtual Memory Allocation Tracking:** Tracing memory page fragmentation boundaries, handling virtual mapping tables, and optimizing page replacement allocation loops.

Studying kernel concurrency mechanisms showed me that high-level data pipelines can easily lock up hardware components if thread contention is ignored.
